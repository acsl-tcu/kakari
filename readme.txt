# 係の引継ぎ資料（LaTeX）

本リポジトリは，係の引継ぎ資料を LaTeX で管理し，
各係ごとの資料を統合して 1 つの PDF として出力するためのものである．  
本フォーマットは 2025年度より GitHub で管理できるように整備された．

---

## ブランチ構成について

本リポジトリでは，年度ごとにブランチを分けて管理する．

- 2024old-version  
  2024年度以前の係引継ぎ資料をまとめたブランチである．  
  過去資料の参照用として保持している．

---

## 係の引継ぎ用 共通ルール（2025年度以降）

2025年度以降は，以下のルールに従って資料を管理すること．

- 各年度ごとに 20××nendo という名前のブランチを作成する．
- 当該年度の係引継ぎ資料は，その年度ブランチ内にまとめる．
- 各係ごとの LaTeX ファイルは `kakari/works` 以下に配置する．
- `main.tex` を実行することで，
  `kakari/works` 内にある各係の TeX ファイルが統合され，
  1 つの PDF として出力される．

---

## 補足

- 過去年度の資料は，原則として編集せず，参照用として保持する．
- 新年度の資料は，必ず新しい年度ブランチを作成して管理すること．

## 注意事項（ファイル名が日本語の場合）

本資料では，ファイル名に日本語を使用している．  
そのため，環境によっては LaTeX のコンパイル時に文字コードの問題が発生し，
正常にコンパイルできない場合がある．

その場合は，ファイル名のエンコードを一時的に euc-jp に変換してからコンパイルすることで回避できる．

---

## Linux でのコンパイル手順（回避策）

以下は Linux 環境での手順例である．  
一時ディレクトリを用いてコンパイルするため，
元のファイル名やデータは変更されない．

```bash
apt-get install convmv
cp -r 係の引き継ぎ資料 tmp
cd tmp
convmv -f utf8 -t euc-jp * --notest
latexmk main
mv main.pdf ../
cd ..
rm -fr tmp
```

※ 一時ファイルを用いてコンパイルしているため，
ファイル名のエンコードを元に戻す必要はない．
